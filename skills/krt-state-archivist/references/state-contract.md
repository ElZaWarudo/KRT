# Compact State Contract

Use this reference when compacting `compound-master-state.md` or reviewing a scaffold generated by `compact_compound_state.py`.

## Purpose

The live state file is a resume entrypoint, not the whole project journal. It should tell the next agent what is true now, where the canonical artifacts live, what is blocked, and what to do next.

The archive holds the full historical record: old verification logs, review rounds, Jira/PR closeouts, merged phase details, repeated brainstorm decisions, and long evidence sections.

## Live State Must Keep

- Initiative, status, mode, date, and production posture when known.
- Source documents and canonical artifact paths.
- Current phase, active work package, and package wave.
- Current branch, integration base, parent branch/stacking strategy, and open PR/Jira items.
- Current blockers or "No blockers".
- Required user decisions.
- Next action and exact next invocation when one exists.
- Links to archived history.
- Any active risk that has not been resolved in a linked artifact.

## Archive Should Receive

- Completed phases and packages.
- Long verification command/result lists.
- Historical review findings that are resolved.
- Merged PR details and closed Jira handoff details.
- Old branch strategies that no longer affect the next action.
- Repeated decisions already captured in brainstorm, plan, or work-package documents.
- CI evidence for completed packages, unless a CI incident remains active.

## Compact State Shape

Prefer this shape:

```markdown
---
title: Compound Master State - <initiative>
status: in_progress
date: YYYY-MM-DD
initiative: <slug>
mode: resume
state_format: compact
last_compacted: YYYY-MM-DD
archive_snapshot: docs/orchestration/archive/compound-master-state/...
---

# Compound Master State - <initiative>

- Next action:

- ...

## Archived History
- Full pre-compaction state: `...`
- Historical sections: ...
```

The exact headings can vary, but the live state must stay easy to scan before resuming work.

## Review Checklist

- The compact file points to the full archive.
- The archive was created before the live state was rewritten.
- No active blocker was archived as historical.
- The next action is explicit enough for `krt-compound-master mode:resume`.
- PR/Jira status reflects open work, not only old closeouts.
- The compact file is materially shorter than the original.
